# 北京时间记录统一修复

## 背景

创建知识库、删除知识库、上传文档、用户管理和审计日志等操作原先主要使用 UTC 时间写入或输出，部分前端页面还依赖浏览器本地时区显示，导致页面展示的操作时间可能不是东八区时间。

## 修复范围

- 后端新增 `app.core.timezone`，统一提供 `beijing_now()`、`as_beijing()`、`isoformat_beijing()`。
- 用户、知识库、文档、聊天、搜索导出等模型的 `created_at` / `updated_at` 默认记录改为北京时间。
- 登录时间、登出撤销时间、上传文件目录日期、仪表板趋势聚合均改为北京时间口径。
- 管理员用户、审计日志、知识库、文档、入库进度、搜索导出、PDF/资产签名等 API 输出统一序列化为 `+08:00`。
- 前端知识库、文档、用户管理、个人中心和聊天会话时间显示固定使用 `Asia/Shanghai`。
- Alembic 配置的版本文件时间改为 `Asia/Shanghai`。
- 新增 Alembic 数据迁移 `beijing_time_records`，对既有业务时间列一次性加 8 小时，补齐历史记录口径；`login_records.expires_at` 保持 UTC，避免影响 JWT 刷新令牌过期校验。

## 说明

JWT 的签发、过期校验仍保留 UTC 作为协议时间基准，仅在 API 返回 `expires_at` 时转换为北京时间显示，避免影响认证安全语义。
